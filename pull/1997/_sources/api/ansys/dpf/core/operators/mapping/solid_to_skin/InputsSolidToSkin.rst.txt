Property detail

.. py:property:: field
   :type: ansys.dpf.core.inputs.Input[ansys.dpf.core.field.Field | ansys.dpf.core.fields_container.FieldsContainer]


   Allows to connect field input to the operator.

   Field containing solid mesh results to map onto the skin surface.

   **Supported locations**:
   - **Elemental**: Values at element centers (mapped to skin element centers)
   - **Nodal**: Values at nodes (rescoped to skin surface nodes)
   - **Elemental Nodal**: Values per element at nodes (extracted for face nodes)

   The field's support mesh provides the solid mesh if pin 2 is not connected. When using a fields container, all fields must have compatible location types and mesh supports.

   :returns: An Input instance for this pin.
   :rtype: input

   .. rubric:: Examples

   >>> from ansys.dpf import core as dpf
   >>> op = dpf.operators.mapping.solid_to_skin()
   >>> op.inputs.field.connect(my_field)
   >>> # or
   >>> op.inputs.field(my_field)

.. py:property:: mesh
   :type: ansys.dpf.core.inputs.Input[ansys.dpf.core.meshed_region.MeshedRegion]


   Allows to connect mesh input to the operator.

   Skin (surface) mesh defining the target 2D boundary representation.

   This mesh must contain topology properties linking it to the solid mesh:
   - **facets**: Property field with face IDs for each skin element
   - **facets_to_ele**: Property field mapping each face to its parent solid element

   The skin mesh is typically generated by a mesh skin extraction operator that automatically creates these topology relationships.

   :returns: An Input instance for this pin.
   :rtype: input

   .. rubric:: Examples

   >>> from ansys.dpf import core as dpf
   >>> op = dpf.operators.mapping.solid_to_skin()
   >>> op.inputs.mesh.connect(my_mesh)
   >>> # or
   >>> op.inputs.mesh(my_mesh)

.. py:property:: solid_mesh
   :type: ansys.dpf.core.inputs.Input[ansys.dpf.core.meshed_region.MeshedRegion]


   Allows to connect solid_mesh input to the operator.

   3D solid (volume) mesh containing the source field data.

   **Optional**: If not provided, the solid mesh is extracted from the field's support (pin 0). Explicitly providing the solid mesh is useful when:
   - The field has no mesh support attached
   - Multiple fields share the same solid mesh (performance optimization)
   - The solid mesh differs from the field's stored support

   The solid mesh must geometrically correspond to the skin mesh, with the skin representing its outer boundary.

   :returns: An Input instance for this pin.
   :rtype: input

   .. rubric:: Examples

   >>> from ansys.dpf import core as dpf
   >>> op = dpf.operators.mapping.solid_to_skin()
   >>> op.inputs.solid_mesh.connect(my_solid_mesh)
   >>> # or
   >>> op.inputs.solid_mesh(my_solid_mesh)
